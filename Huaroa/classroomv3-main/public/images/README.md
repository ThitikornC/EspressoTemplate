# 📁 Image Assets Guide

## โครงสร้างโฟลเดอร์รูปภาพ

```
public/images/
├── coloring/        # รูปสำหรับเกมระบายสี
├── puzzle/          # รูปสำหรับเกมจิ๊กซอว์
├── matching/        # รูปสำหรับเกมจับคู่
└── shadow/          # รูปสำหรับเกมจับเงา
```

## วิธีเพิ่มรูปภาพเข้าโปรเจกต์

### 1. สำหรับเกมระบายสี (Coloring)
- วางไฟล์รูปใน: `public/images/coloring/`
- ตั้งชื่อไฟล์: `image-1.png`, `image-2.png`, `image-3.png` ...
- รูปแบบที่รองรับ: PNG, JPG, WEBP
- ขนาดแนะนำ: 800x600 หรือสัดส่วนที่พอดี
- **คำแนะนำ**: ใช้รูปเส้นขอบ (line art) หรือรูปโครงสร้างที่ชัดเจน

### 2. สำหรับเกมจิ๊กซอว์ (Puzzle)
- วางไฟล์รูปใน: `public/images/puzzle/`
- ตั้งชื่อไฟล์: `image-1.png`, `image-2.png`, `image-3.png` ...
- รูปแบบที่รองรับ: PNG, JPG, WEBP
- ขนาดแนะนำ: 600x400 หรือ 800x600
- **คำแนะนำ**: ใช้รูปที่มีรายละเอียดชัดเจน ไม่มีพื้นหลังซับซ้อนเกินไป

### 3. สำหรับเกมจับคู่ (Matching)
- วางไฟล์รูปใน: `public/images/matching/`
- ตั้งชื่อไฟล์: 
  - `left-1.png`, `right-1.png` (คู่ที่ 1)
  - `left-2.png`, `right-2.png` (คู่ที่ 2)
  - หรือใช้ `pair-1-a.png`, `pair-1-b.png`
- รูปแบบที่รองรับ: PNG, JPG, WEBP
- ขนาดแนะนำ: 100x100 หรือ 150x150 (รูปสี่เหลี่ยม)
- **คำแนะนำ**: ใช้รูปที่เห็นความแตกต่างชัดเจน

### 4. สำหรับเกมจับเงา (Shadow)
- วางไฟล์รูปใน: `public/images/shadow/`
- ตั้งชื่อไฟล์:
  - `object-1.png` (วัตถุจริง)
  - `shadow-1.png` (เงา)
- รูปแบบที่รองรับ: PNG (แนะนำใช้ PNG เพื่อพื้นหลังโปร่งใส)
- ขนาดแนะนำ: 150x150 หรือ 200x200
- **คำแนะนำ**: 
  - วัตถุจริงใช้สีสดใส
  - เงาควรเป็นสีดำหรือเทาเข้ม

## ตัวอย่างการใช้งาน

### เพิ่มรูปเข้า Coloring Game:
1. บันทึกรูปเป็น `image-1.png`
2. วางใน `public/images/coloring/`
3. เปิดเกมระบายสี จะเห็นรูปใหม่ในรายการ

### การแก้ไข Config (ถ้าต้องการเพิ่มรูปเพิ่มเติม):
แก้ไขไฟล์ `src/config/defaultImages.ts`:

```typescript
export const defaultColoringImages: ImageItem[] = [
  {
    id: 'coloring-1',
    name: 'รูประบายสี 1',
    url: '/images/coloring/image-1.png',
    category: 'coloring'
  },
  // เพิ่มรูปใหม่ที่นี่
  {
    id: 'coloring-4',
    name: 'รูประบายสี 4',
    url: '/images/coloring/image-4.png',
    category: 'coloring'
  }
]
```

## ขนาดและรูปแบบไฟล์

### ขนาดไฟล์แนะนำ:
- **ไม่เกิน 2MB** ต่อไฟล์
- ใช้ compression เพื่อลดขนาดไฟล์
- แนะนำ: [TinyPNG](https://tinypng.com/) สำหรับ compress

### รูปแบบไฟล์:
- ✅ **PNG** - แนะนำสำหรับรูปที่ต้องการพื้นหลังโปร่งใส
- ✅ **JPG/JPEG** - แนะนำสำหรับรูปถ่าย
- ✅ **WEBP** - ขนาดเล็ก คุณภาพดี (modern browsers)
- ❌ **GIF** - ไม่แนะนำ (ใช้ได้แต่ขนาดใหญ่)

## Tips & Best Practices

### 🎨 สำหรับรูประบายสี:
- ใช้เส้นขอบสีดำหนา 2-3px
- พื้นหลังสีขาวหรือโปร่งใส
- หลีกเลี่ยงรายละเอียดเล็กเกินไป
- ใช้ภาพที่เด็กๆ คุ้นเคย (สัตว์, ผลไม้, ยานพาหนะ)

### 🧩 สำหรับจิ๊กซอว์:
- ใช้รูปที่มีสีสันสดใส
- หลีกเลี่ยงพื้นหลังสีเดียว
- มีจุดเด่นที่ชัดเจน
- ไม่ซับซ้อนเกินไป

### 🎯 สำหรับจับคู่:
- รูปควรเห็นความแตกต่างชัดเจน
- ขนาดเท่ากันทุกรูป
- พื้นหลังสีเดียวกัน
- ใช้ไอคอนหรือรูปง่ายๆ

### 👥 สำหรับจับเงา:
- เงาควรมีรูปร่างตรงกับวัตถุ
- พื้นหลังโปร่งใส (PNG)
- สีเงาเป็นดำ 70-80% opacity
- วัตถุและเงาแยกไฟล์กัน

## 🚨 Troubleshooting

### รูปไม่แสดง:
1. ตรวจสอบชื่อไฟล์ว่าตรงกับ config หรือไม่
2. ตรวจสอบตำแหน่งโฟลเดอร์
3. รีเฟรชเบราว์เซอร์ (Ctrl+F5)
4. ตรวจสอบ console ใน DevTools

### รูปโหลดช้า:
1. ลดขนาดไฟล์ด้วย compression
2. ใช้ WEBP แทน PNG/JPG
3. ปรับขนาดรูปให้เหมาะสม

### รูปเพี้ยน:
1. ตรวจสอบ aspect ratio
2. ใช้ขนาดที่แนะนำ
3. ตรวจสอบ CSS ที่เกี่ยวข้อง
